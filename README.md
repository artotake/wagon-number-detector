Описание ячеек проекта

Ячейка 1 — Подготовка окружения
  Импортирует все нужные библиотеки (для работы с изображениями, нейросетями, математикой) и подключает Google Диск, где лежат данные.

Ячейка 2 — Разделение данных на train и test
  Находит все файлы (изображения и аннотации)
  Делит их на 70% для обучения и 30% для проверки
  По умолчанию использует только 1000 изображений (чтобы быстро обучать на обычном компьютере). Для полного обучения нужно изменить переменную USE_N_FILES = None.

Ячейка 3 — Загрузчик данных (Dataset)
  Загружает одно изображение и его аннотацию (текстовый файл с координатами цифр)
  Преобразует координаты в нужный формат
  Применяет аугментации (если нужно)
  Возвращает пару: изображение + разметку (где какие цифры)

Ячейка 4 — Аугментации и DataLoader'ы
  Определяет, как изменять изображения во время обучения (повороты, изменение яркости — это помогает модели не переобучаться)
  Создаёт загрузчики данных (train_loader и test_loader), которые подают данные в модель пачками (batch'ами)

Ячейка 5 — Создание модели
  Создаёт нейросеть Faster R-CNN на базе MobileNetV3 (это популярный детектор объектов, адаптированный для мобильных устройств)
  Настраивает оптимизатор (AdamW) и планировщик скорости обучения
  Использует собственную архитектуру

Ячейка 6 — Функция для расчёта mAP
  Берёт предсказания модели и сравнивает их с правильной разметкой
  Вычисляет метрику mAP (mean Average Precision) — главный показатель качества детекции
  Считает точность отдельно для каждой цифры (от 0 до 9)

Ячейка 7 — Функции обучения и валидации
  train_one_epoch — обучает модель одну эпоху (показывает, как уменьшается ошибка)
  validate — проверяет модель на тестовых данных без обучения

Ячейка 8 — Запуск обучения
  Запускает цикл обучения на 5 эпох
  Сохраняет лучшую модель (с наименьшей ошибкой на валидации)
  Использует раннюю остановку — если ошибка долго не улучшается, обучение прекращается
  При полном датасете нужно увеличить количество эпох до 50–100

Ячейка 9 — Оценка качества (mAP)
  Загружает лучшую сохранённую модель
  Запускает расчёт mAP на тестовой выборке
  Выводит общую точность и точность по каждой цифре
  
Ячейка 9.5 — Загрузка модели для визуализации
  Пересоздаёт модель и загружает в неё сохранённые веса
  Готовит модель к показу предсказаний

Ячейка 10 — Визуализация предсказаний
  Показывает несколько примеров из тестовой выборки
  Зелёные рамки — правильное расположение цифр (ground truth)
  Красные рамки — то, что предсказала модель
  Позволяет визуально оценить, насколько хорошо модель научилась находить цифры


Как запустить проект
  Открыть в Google Colab (кнопка "Open in Colab")
  Смонтировать Google Диск (в ячейке 1)
  Загрузить датасет на Диск по пути: My Drive/test_task/digits/up+low
  Запустить все ячейки по порядку (Shift + Enter)

Ожидаемые результаты
  После обучения: ошибка (loss) постепенно уменьшается
  После оценки: mAP около 0.7–0.9 на полном датасете
  Визуализация: красные рамки должны примерно совпадать с зелёными

Примечание
  По умолчанию проект использует 1000 изображений для быстрой проверки. Для получения реальных результатов:
  Установите USE_N_FILES = None в ячейке 2

Увеличьте num_epochs до 50–100 в ячейке 8

Используйте GPU (Среда выполнения → Сменить среду выполнения → T4 GPU)
